5. Activities & Logic: The Heart of P6
Activity Types
Task Dependent
Default. Use for normal work. Calendar based on project.
Resource Dependent
Use when crew size matters. Calendar based on resource.
Level of Effort (LOE)
For ongoing support (PM, QA). Spans predecessors/successors.
Milestone
Start/Finish. Duration = 0. Use for key events.
WBS Summary
Rollup durations/costs from child activities.
Add Activity
  1. Go to Activities view
  2. Press Insert
  3. ID: A1000 (auto or manual)
  4. Name: Submit Design Package
  5. Duration: 5d
  6. Assign to WBS: 1.1
  7. Add relationships: Predecessors/Successors

  5. Budgeted Units: 40h
  6. Set curve: Linear/Front-loaded
Resource Allocation Best Practices & How-To
  • Avoid Overallocation: Use Tools ‚Üí Level Resources to resolve peaks.
  • Roles First: Assign roles, then replace with resources for flexibility.
  • Curves: Use resource curves for non-linear distribution (e.g., bell curve for ramp-up).
  • Shifts/Calendars: Assign resource-specific calendars for accurate availability.
  • Thresholds: Set max units to prevent unrealistic assignments.
Golden Rule: Always assign Roles first, then replace with actual Resources later. Monitor usage spreadsheets.
üí° Tip: Use Resource Usage Profile to visualize allocation. Adjust lags for smoothing.
7. Scheduling Engine & Critical Path Best Practices
Schedule Project (F9)
  1. Press F9
  2. Check Log to File
  3. Set Data Date
  4. Click Schedule
  5. Review log for errors
Scheduling Best Practices & How-To
  • Logic Ties: Use FS mostly, SS/FF with lags sparingly. Avoid FF for cycles.
  • Constraints: Minimize hard constraints; use soft (expected finish).
  • Data Date: Update weekly; progress activities before scheduling.
  • Calendars: Use global/project calendars consistently for holidays/weather.
  • Float Management: Monitor total float; buffer critical paths.
Critical Path
  • Activities with Total Float ‚â§ 0
  • Shown in red by default
  • Filter: View ‚Üí Filters ‚Üí Critical
Critical Path Best Practices & How-To
  • Identify Multiple Paths: Use longest path option in schedule settings.
  • Buffer: Add contingency activities on critical path.
  • Monitor: Weekly review variance from baseline.
  • Optimize: Crash activities by adding resources, but calculate cost impact.
  • Avoid Artificial Criticality: Remove unnecessary constraints.
Never have open ends! Every activity must have a predecessor and successor (except milestones). Check for loops.
üí° Tip: Use Trace Logic view to debug critical path. Set critical float threshold in preferences.
