5. Activities & Logic: The Heart of P6

Activities are the actionable steps within your WBS. Proper logic ties them together in sequence. The network of relationships determines the project‚Äôs critical path and overall scheduling accuracy.

Activity Types
Task Dependent
Default. Uses activity calendar. Ideal for straightforward tasks.
Resource Dependent
Uses resource calendar. Helpful when resource availability drives schedule.
Level of Effort (LOE)
Spans between predecessor and successor. Common for project admin tasks.
Milestone
Zero-duration. Marks key events or phase gates.
WBS Summary
Rolls up durations/costs from child activities within a WBS element.

  6. Assign to WBS: 1.1

  8. Set constraints (optional, use sparingly)
üí° Naming Convention: [Verb] + [Noun] ‚Üí "Install Rebar", "Pour Foundation". Keep IDs sequential. Use activity codes to group or filter later. Clarity in naming reduces confusion among your team.

  3. Click Add Resource

  6. Choose a resource curve (linear, front-loaded, etc.) if needed
Resource Allocation Best Practices & How-To
  • Avoid Overallocation: Use Tools ‚Üí Level Resources to resolve peaks.
  • Roles First: Assign roles, then replace with named resources for real-world scheduling flexibility.
  • Curves: Consider bell curves or front-loaded distribution if work ramps up or tapers off.
  • Shifts/Calendars: Align resource calendars with actual working shifts/availability.
  • Thresholds: Set realistic max units/time; never schedule a single laborer for 24 hours a day.
Golden Rule: Always assign Roles first, then replace with actual Resources once you confirm availability. Leverage usage spreadsheets to validate workloads.
üí° Tip: Use Resource Usage Profile to visually spot overallocations. Adjust lags or schedule constraints to even out resource spikes. Also, incorporate real rates and overhead costs to get more accurate cost forecasting.

7. Scheduling Engine & Critical Path Best Practices

Primavera P6‚Äôs scheduling engine calculates start and finish dates based on your activity relationships, calendars, and constraints. Managing the critical path is crucial for on-time project delivery and guiding management decisions.

Schedule Project (F9)
  1. Press F9 or go to Tools ‚Üí Schedule
  2. Enable Log to File to capture errors or warnings
  3. Set Data Date correctly (e.g., current day or last update date)
  4. Click Schedule
  5. Review log for open ends, negative float, or resource conflicts
Scheduling Best Practices & How-To
  • Logic Ties: Prefer FS relationships. Use SS/FF relationships sparingly to avoid confusion.
  • Constraints: Keep ‚ÄúStart On‚Äù or ‚ÄúMust Finish On‚Äù to a minimum. Overusing constraints can distort the critical path.
  • Data Date: Update weekly or monthly; always progress your activities before re-running schedule.
  • Calendars: Use consistent project-specific calendars for holidays or weather contingencies.
  • Float Management: Track total float to mitigate schedule risk and identify near-critical paths.
Critical Path
  • Activities with Total Float ‚â§ 0
  • By default, displayed in red.
  • Use View ‚Üí Filters ‚Üí Critical or customize to fit zero/negative float thresholds.
Critical Path Best Practices & How-To
  • Identify Multiple Paths: Set ‚ÄúCalculate multiple float paths‚Äù in schedule settings when needed.
  • Buffer: Insert contingency tasks or additional resource availability along the critical path to protect deadlines.
  • Monitor: Compare baselines weekly or monthly to catch slippage early.
  • Optimize: If needed, resource ‚Äúcrash‚Äù or fast-track, but weigh cost/scope trade-offs thoroughly.
  • Avoid Artificial Criticality: Don‚Äôt artificially constrain tasks unless absolutely required.
Never have open ends! Ensuring each activity has a valid predecessor and successor (except the first and last milestone) prevents scheduling errors and spurious float values.
üí° Tip: Use ‚ÄúTrace Logic‚Äù to visualize predecessor-successor chains. Adjust the critical float threshold in Primavera‚Äôs schedule options for near-critical path analysis.

Handling Lag & Lead:
- Use leads (negative lag) cautiously, as they can confuse project teams.
- Document any complicated logic for clarity in scheduling reviews.
- Thoughtful scheduling clarifies resources, deadlines, and risk areas.
