Primavera P6 Comprehensive Guide

Key Vocabulary & Acronyms

Below is a quick reference to acronyms, terms, and definitions commonly used in project scheduling and Primavera P6.

  • WBS (Work Breakdown Structure): A hierarchical decomposition of the total scope of work.
  • EPS (Enterprise Project Structure): The top-level hierarchical structure in P6 that organizes multiple projects.
  • OBS (Organizational Breakdown Structure): Defines the company’s management structure for resource responsibilities.
  • Activity: A specific task or set of tasks carried out within a project.
  • Critical Path: The sequence of activities that determines the earliest completion date of a project.
  • Baseline: A snapshot of the project’s original plan used to measure progress.
  • Constraint: A date restriction or limitation placed on an activity’s start or finish.
  • Resource: Labor, material, equipment, or cost assigned to activities.
  • Duration: The total span of working time required to complete an activity.
  • Float: The amount of time an activity can be delayed without impacting the project’s finish date.
  • Predecessor/Successor: Activities that come before (predecessor) or after (successor) a given activity in a schedule.
  • Gantt Chart: A bar chart that illustrates a project schedule.

History of Scheduling

Early Beginnings

Project scheduling traces back to the early 20th century with the development of the Gantt Chart. Named after Henry Gantt, this chart revolutionized how tasks and timeframes were visualized, enabling managers to plan, schedule, and control work more efficiently.

PERT and CPM

In the 1950s, the Program Evaluation and Review Technique (PERT) and the Critical Path Method (CPM) emerged. These methods introduced the concept of dependencies and the idea that certain tasks "drive" the project completion date—creating a “critical path.”

Digital Transformation

With the arrival of computers, scheduling advanced into software solutions—making it easier to handle large, complex projects. Microsoft Project, OpenPlan, and Primavera were among the pioneers of computerized scheduling tools, each offering unique features but similar underlying principles for defining tasks, connecting dependencies, and calculating schedules.

Primavera P6 History

Primavera Systems, Inc. was founded in 1983. Their Primavera P3 software quickly became a standard in the construction and engineering industries. Over time, the tool evolved into Primavera P6 (originally introduced as “Primavera Project Planner for the Enterprise”). Oracle acquired Primavera in 2008, and since then, P6 has continued to grow in power and functionality, becoming one of the most widely used project portfolio management solutions worldwide.

Comparing P6 to Other Planning Software

Pros

  • Scalability: P6 handles large, complex projects with thousands of activities without major performance issues.
  • Resource Management: Detailed resource assignments and leveling features.
  • Enterprise-Level Integration: Supports multi-project environments through EPS and OBS structures.
  • Robust Reporting: Advanced reporting, baseline comparison, and tracking features.
  • Global Change: Automates bulk modifications across multiple activities.

Cons

  • Learning Curve: The interface and wide range of features can be overwhelming to newcomers.
  • Cost: Licensing and implementation can be expensive compared to smaller-scale tools.
  • User Experience: Some find the UI less intuitive compared to simpler software like Microsoft Project.
  • Integration Complexity: Incorporating P6 with other systems may require specialized IT support.

Ideal Use Cases

Primavera P6 is best suited for large-scale, complex projects or when multiple projects are managed under the same organization with shared resources. It’s particularly popular in construction, oil and gas, engineering, and infrastructure development.

Creating a Schedule in P6

Step-by-Step Overview

  1. Create EPS Node: Configure an EPS node under which you’ll create the new project.
  2. Set Project Properties: Enter the project start date, default calendar, and other settings.
  3. Build Your WBS: Define the high-level structure of project deliverables.
  4. Add Activities: Within each WBS element, add activities (tasks), durations, and descriptions.
  5. Assign Relationships: Link activities with predecessors and successors to form the logical sequence.
  6. Assign Resources: Specify the labor, material, and equipment needed for each activity.
  7. Define Calendars: Use global or project calendars to set working and non-working days.
  8. Schedule (Calculate): Use the “Schedule” function to calculate dates and float.
  9. Add Baseline (Optional): Create a baseline once the plan is finalized.

Once complete, your project plan will display start/finish dates for all tasks and highlight the critical path.

Maintaining a Schedule in P6

Progress Updates

Collect actual start dates, updating completed percentages and resource usage. Regularly “schedule” again to produce revised dates, float margins, and to pinpoint any at-risk activities.

Tracking & Reporting

Run or customize in-built reports using the P6 Reporting tools. Compare progress to the baseline. Identify any variances to help with corrective actions.

Managing Changes & Re-baselining

If the project scope changes, apply the modifications to the schedule. Consider updating or creating a new baseline to maintain a clear record of planned vs. actual.

Types of Constraints
  • Start On/After/Before: Imposes a restriction on the earliest/latest start.
  • Finish On/After/Before: Imposes a restriction on the earliest/latest finish.
  • Mandatory Start/Finish: Locks the activity to a specific date, overriding calculations.
  • As Late As Possible (ALAP): Places the activity so it ends right before its successor begins.

Constraints can ensure certain deadlines are met or that tasks align with real-world events, but overuse can hide true critical paths. Use constraints with caution.

Tracing the Critical Path

Methodology

The critical path is the chain of activities with zero float, meaning any delay directly affects project completion. P6 automatically calculates the critical path once you schedule the project.

How To Identify Critical Activities

  1. Open the Project: Go to your project’s Gantt chart.
  2. Press F9 or Click “Schedule”: Recalculate the schedule (or “Schedule”).
  3. Filter or Highlight: Apply the “Critical” filter or highlight the critical path in the Gantt chart to see all zero-float activities.

Managing the Critical Path

Focus on these activities to ensure the project stays on track. If they slip, the overall completion date slips. Evaluate if any can be fast-tracked (done in parallel) or crashed (resources increased to reduce duration).
